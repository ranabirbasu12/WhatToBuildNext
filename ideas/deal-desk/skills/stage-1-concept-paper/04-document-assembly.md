# Stage 1.4: Document Assembly

You are assembling the final Concept Paper from the outputs of the preceding sub-skills. This is the integration step — turning thesis pillars, deal terms, and risk analysis into a cohesive IC presentation.

---

## The Canonical Section Order

Based on Casablanca (16 slides), MPokket (19 pages), and Asandas (20 pages), every Ascertis Concept Paper follows this structure:

| # | Section | Pages/Slides | Source Sub-Skill |
|---|---------|-------------|------------------|
| 1 | **Concept Paper Checklist** | 1 | All (synthesis) |
| 2 | **Executive Summary** | 1 | Thesis Builder |
| 3 | **Opportunity Overview** | 1-2 | Thesis Builder + Stage 0 |
| 4 | **Deal Terms Overview** | 1-2 | Structure Drafter |
| 5 | **Corporate Structure Diagram** | 1 | Structure Drafter |
| 6 | **Transaction Structure** | 1 | Structure Drafter |
| 7 | **Business Profile** | 2-3 | Stage 0 data + Thesis Builder |
| 8 | **Promoter / Management** | 1-2 | Thesis Builder |
| 9 | **Industry Context** | 1-2 | Stage 0 + external research |
| 10 | **Financial Performance** | 2-3 | Stage 0 financials |
| 11 | **Investment Thesis** | 1-2 | Thesis Builder |
| 12 | **Risks** | 1-2 | Risk Framework |
| 13 | **ESG Assessment** | 0.5-1 | Risk Framework |
| 14 | **Use of Proceeds** | 0.5-1 | Structure Drafter |
| 15 | **Next Steps / DD Plan** | 1 | Risk Framework |
| 16 | **Appendices** (if needed) | Variable | Various |

**Total: 15-25 pages.** The IC reads this in advance. The deal lead presents the highlights in 15-20 minutes.

## Section-by-Section Assembly Guide

### Section 1: Concept Paper Checklist (MOST IMPORTANT)

This is Slide 1 / Page 1. The IC looks at this first.

```
CONCEPT PAPER CHECKLIST

Company: [Name] | Project Code: [Codename]
Date: [DD/MM/YYYY] | Deal Lead: [Name] | Associate: [Name]

| # | Dimension                          | Assessment    | Key Comment                                |
|---|-----------------------------------|---------------|-------------------------------------------|
| 1 | Sponsor / Business Quality        | Yes / No / TBD | [One line on promoter + business moat]    |
| 2 | Financial Health                  | Yes / No / TBD | [One line on leverage, trajectory, CF]     |
| 3 | Industry & Competitive Dynamics   | Yes / No / TBD | [One line on market position + trends]     |
| 4 | Transaction Structure             | Yes / No / TBD | [One line on structural cleanliness]       |
| 5 | Yield                             | Yes / No / TBD | [One line on target IRR achievability]     |

Recommendation: PROCEED TO TERM SHEET + DD / REJECT
```

Rules:
- If ANY dimension is "No" → must have a strong justification for still recommending the deal
- "TBD" is acceptable at CP stage — it becomes a DD resolution item
- Every "TBD" must map to a DD workstream in the Next Steps section

### Section 2: Executive Summary

One page. The entire CP distilled into 4-5 paragraphs:

```
Paragraph 1: The opportunity (who, what, how much, what instrument)
Paragraph 2: Why this company (2-3 strongest thesis pillars in one sentence each)
Paragraph 3: Deal terms summary (size, tenor, coupon, security, target IRR)
Paragraph 4: Key risks and mitigants (top 2-3 risks, one line each)
Paragraph 5: Recommendation and next steps
```

This page must be self-contained. An IC member who reads ONLY this page should understand the deal.

### Section 3-4: Opportunity & Deal Terms

**Opportunity Overview:**
- Company name, sector, geography
- What the company does (1-2 sentences — clear, not marketing)
- Why they need capital (acquisition, capex, refinance, growth)
- Why now (what triggered this deal opportunity)

**Deal Terms Overview:**
Pull directly from Structure Drafter output. Present as a clean table:

| Parameter | Proposed Terms |
|-----------|---------------|
| Facility | INR ___ Cr Senior Secured NCDs |
| Tenor | ___ months |
| Coupon | ___% p.a. (fixed, quarterly) |
| Additional Return | Accrued premium ___% + Upside premium ___% |
| Security | [1-2 line summary] |
| Target IRR | ___% |

### Section 5-6: Structure Diagrams

Include both diagrams from Structure Drafter:
1. **Corporate structure** — who owns what, percentage stakes
2. **Transaction structure** — flow of funds from fund vehicle to operating entity

These must be visual. Use org-chart style diagrams. The IC should understand ownership and fund flow at a glance.

Flag structural risks directly on the diagram (annotate with callout boxes).

### Section 7-8: Business Profile & Management

**Business Profile:**
- Business description (what they make/do, how they make money)
- Products/services breakdown (% of revenue by segment)
- Customer base (top 5 customers, concentration)
- Capacity and operations (plants, locations, utilization)
- Competitive positioning (market share, ranking)

**Promoter / Management:**
- Promoter background (track record, prior ventures, net worth if available)
- Key management team (name, role, background — 3-5 key people)
- Board composition (independent directors, institutional nominees)
- Governance notes (family involvement, professional management depth)

### Section 9: Industry Context

Brief industry overview — NOT a full industry note (that's Stage 3). Just enough to:
- Size the market (TAM, growth rate)
- Identify 2-3 tailwinds / headwinds
- Position the company relative to peers
- Flag industry-specific risks

For NBFC deals: Include regulatory landscape (RBI framework, recent guidelines).
For agri deals: Include seasonality, commodity dynamics, trade policy.

### Section 10: Financial Performance

Pull from Stage 0 BPEA-format financials. Present:

**Income Statement Summary (3-5 years):**
```
| (INR Cr) | FY__ | FY__ | FY__ | FY__ | FY__E |
|----------|------|------|------|------|-------|
| Revenue  |      |      |      |      |       |
| EBITDA   |      |      |      |      |       |
| EBITDA % |      |      |      |      |       |
| PAT      |      |      |      |      |       |
```

**Key Credit Metrics:**
```
| Metric           | FY__ | FY__ | FY__ |
|------------------|------|------|------|
| Net Debt/EBITDA  |      |      |      |
| ISCR             |      |      |      |
| DSCR             |      |      |      |
| Current Ratio    |      |      |      |
| Net WC Days      |      |      |      |
```

**Balance Sheet Snapshot:**
- Net worth, total debt, net debt
- Fixed assets, current assets
- Working capital cycle (debtor days, inventory days, creditor days)

For NBFC: Replace with AUM growth, NIM, credit costs, DuPont decomposition.

### Section 11: Investment Thesis

Present 4-6 pillars from Thesis Builder. Each pillar gets:
- **Header** (one line — "Experienced promoter with proven exit track record")
- **Evidence** (2-3 bullet points with specific data)
- **Confidence flag** (implicit — audited data vs. management representations)

### Section 12-13: Risks & ESG

Pull from Risk Framework. Present:
- **Risk summary table** (5-8 risks with category, severity, one-line description)
- **Top 3 risks expanded** (paragraph-level discussion for Critical/Investigate risks)
- **ESG classification** (Category A/B/C, applicable IFC standards, key issues)

### Section 14: Use of Proceeds

Clean table from Structure Drafter showing where every rupee goes.

### Section 15: Next Steps

This is the action plan if IC approves:

```
| # | Action | Vendor / Owner | Timeline | Connected Risk |
|---|--------|---------------|----------|----------------|
| 1 | Financial DD | [Deloitte/EY] | 4-6 weeks | Financial risks |
| 2 | Legal DD | [Trilegal/TT&A] | 4-6 weeks | Structural risks |
| 3 | Commercial DD | [EY Parthenon] | 4-6 weeks | Business risks |
| 4 | ESG DD | [EGP Partners] | 3-4 weeks | ESG risks |
| 5 | Background DD | Internal | 2-3 weeks | Promoter risk |
| 6 | GLG expert calls | GLG | 2-3 weeks | Industry risks |

| 8 | Plant/site visit | Deal team | 2-3 weeks | Operations validation |

```

Every DD action must connect back to a risk identified in Section 12. This closes the loop: risk identified → DD will resolve it → here's the timeline and vendor.

## Format Guidelines

### PowerPoint Format (Traditional)
- One concept per slide
- Heavy use of tables, charts, diagrams
- Minimal text — bullet points, not paragraphs
- Title slide with project codename + date + deal team
- Font: consistent (typically Calibri or Arial)
- Colors: fund branding

### Word Document Format (Recent Shift — Asandas)
- Numbered sections and sub-sections
- More narrative text allowed
- Track changes enabled for review cycles
- Section breaks between major topics
- Header/footer with project code + version + date + page number

### Charts to Include
Based on Asandas's "Charts for Concept Paper.pptx":
- Revenue and EBITDA bar chart (3-5 year trend)
- Leverage trajectory chart (entry → exit)
- Market share / competitive positioning
- Capacity utilization trend
- Customer concentration pie chart

## Review Cycle Management

After first draft assembly, the review cycle begins:

### Round 1: Self-Review
Before sending to deal lead, check:
- [ ] Checklist is filled — no blanks
- [ ] Executive summary is self-contained
- [ ] Every thesis pillar has evidence with source
- [ ] Every risk has a DD action
- [ ] Numbers reconcile between financial tables and text references
- [ ] Structure diagram matches terms table
- [ ] No typos in company name or project codename (common: "Crsip" → "Crisp")
- [ ] Use of proceeds adds up to facility size
- [ ] Target IRR calculation is defensible

### Round 2: Deal Lead (SM/VP)
Expect 10-30 comments. Common feedback patterns:
- "Strengthen evidence on Pillar X" → add specific data points
- "This risk is understated" → elevate severity and add mitigation detail
- "Terms need work" → adjust based on deal lead's market knowledge
- "Industry section is thin" → add peer comparison or market data

### Round 3: CIO (KG) Review
Strategic-level feedback:
- "Why are we doing this deal?" → thesis pillar challenge
- "Entry leverage is uncomfortable" → need stronger deleveraging evidence
- "This structure is too complex" → simplify or flag as deal-breaker
- Expect a redline version with tracked changes

### Round 4+: Final Clean-Up
- Incorporate all redline comments
- Produce clean version (no tracked changes)
- Export PDF for IC distribution
- Prepare 15-minute verbal summary for presentation

## Output

The document assembly produces:
1. **Complete Concept Paper** (15-25 pages, all sections populated)
2. **Supporting charts file** (if PowerPoint format, separate charts deck)
3. **Review tracking log** (comments received, how addressed)

This is the final Stage 1 deliverable. Upon IC approval, the deal moves to:
- **Stage 2 (HoT)** — term sheet issuance and negotiation
- **Stage 3 (Industry Note)** — deep industry research via GLG expert calls
- **Stage 4 (DD File)** — comprehensive due diligence begins
